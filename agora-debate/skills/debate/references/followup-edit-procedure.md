# 후속 토론 결과 Edit 절차

원본 토론 결과 파일의 각 섹션을 Edit 도구로 직접 수정합니다.

## 후속 토론 번호(#N) 결정
- 토론 이력 테이블의 행 수를 세어 다음 번호 결정
- 최초 토론만 있으면 `후속#1`, 후속#1이 있으면 `후속#2`, ...

## Edit 절차 (7단계)

### Edit 1: 토론 이력 테이블에 행 추가
`### 토론 이력` 테이블의 마지막 행 뒤에 새 행 추가:
```
| 후속#[N] | [YYYY-MM-DD HH:mm] | [후속 토론 초점] | [참여자 목록] | [라운드]회 | [종료 사유] |
```

### Edit 2: 합의된 사항 업데이트
`### 합의된 사항` 섹션에서:
- 후속 토론에서 새로 합의된 항목 추가 (번호 이어서)
- 미합의→합의로 전환된 항목 추가 + `[후속#N에서 합의]` 태그

예시:
```markdown
### 합의된 사항
1. 단계적 도입 필요
2. 교육 프로그램 선행
3. 3분기 내 파일럿 실시 `[후속#1에서 합의]`
```

### Edit 3: 미합의 사항 업데이트
`### 미합의 사항` 섹션에서:
- 합의로 전환된 항목 삭제
- 새로 제기된 미합의 항목 추가 + `[후속#N에서 신규 제기]` 태그

예시:
```markdown
### 미합의 사항
1. 파일럿 대상 부서 `[후속#1에서 신규 제기]`
```

### Edit 4: 핵심 인사이트 추가
`### 핵심 인사이트` 섹션 끝에 새 인사이트 추가 + `[후속#N]` 태그:

예시:
```markdown
### 핵심 인사이트
- 기술과 인력의 균형 필요
- 소규모 파일럿으로 리스크 최소화 `[후속#1]`
```

### Edit 5: 추가 검토 권장 사항 업데이트
`### 추가 검토 권장 사항` 섹션에서:
- 후속 토론으로 해결된 항목 삭제
- 새 권장 사항 추가 + `[후속#N]` 태그

### Edit 6: 기존 논쟁 상태 업데이트
`## 주요 논쟁` 섹션의 기존 논쟁 블록에서:
- **현재 상태** 행을 업데이트 + `[후속#N]` 태그

예시:
```markdown
### 즉시 도입 vs 단계적 도입
- **찬성 측**: 경쟁력 확보 (낙관론자)
- **반대 측**: 준비 부족 (비관론자)
- **현재 상태**: 합의 - 3분기 파일럿으로 결론 `[후속#1]`
```

### Edit 7: 새 논쟁 블록 추가
`## 주요 논쟁` 섹션 끝에 후속 토론에서 새로 등장한 논쟁 추가 + `[후속#N]` 태그:

예시:
```markdown
### 파일럿 대상 부서 선정 `[후속#1]`
- **찬성 측**: IT부서 우선 (혁신가)
- **반대 측**: 영업부서 우선 (현실주의자)
- **현재 상태**: 미합의
```
